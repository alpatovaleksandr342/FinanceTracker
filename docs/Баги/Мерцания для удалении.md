## Причина:
``` tsx
if (confirm("Удалить услугу? ")) { 
deleteService.mutate({ id: id }); 
}
```
 Нативный confirm открывается поверх страницы, по этому при подтверждении в окне, заново происходил фокус и запрос данных.
 Так как я использую оптимистичное обновление данных при изменении происходило следующее:
 Вызов удаления -> поле пропадает в ui(оптимистично) -> происходит фокус -> поле все ещё в базе -> появляется -> поле удаляется из базы -> обновление данных -> поле удаляется из ui
 
## Решение
 Подключил @mantine/modals которые не вызывают расфокус страницы
 Протестировал внутри компонента -> работает как надо
 Написал универсальное окно подтверждения:
 
 ``` tsx
export const confirmModal = (title: string, onConfirm: () => void) => {
return modals.openConfirmModal({
title: title,
labels: { confirm: "Да", cancel: "Нет" },
onConfirm: () => onConfirm(),
});
};
 ```
 Начал использовать в функции:
``` tsx
const handleDelete = async (id: string) => {
confirmModal("Вы действительно хотите удалить эту услугу?", () => deleteService.mutate({ id: id }));
};
```
## Почему именно так?
я мог воспользоваться параметрами запроса страницы
``` tsx
const servicesList = trpc.services.getAll.useQuery(undefined, {
  refetchOnWindowFocus: false,
});
```
 Но в таком случае теряется часть отзывчивости страницы и перестают самостоятельно синхронизироваться данные с базой
