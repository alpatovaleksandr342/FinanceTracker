## До оптимизации

```tsx
// useWorkers.ts — состояние модалки в хуке
const [opened, { open }] = useDisclosure();
const [editingWorker, setEditingWorker] = useState();

// Любое изменение → ререндер WorkersPage → ререндер всей таблицы
```
## После оптимизации
``` tsx
// modalStore.ts — состояние изолировано
export const useModalStore = create((set) => ({
  editingWorker: null,
  isOpen: false,
  openModal: (w) => set({ editingWorker: w, isOpen: true })
}));

// WorkersPage — подписка только на функции
const openModal = useModalStore(s => s.openModal);
```

## Что было сделано?

Хранение состояния модального окна было вынесено с кастомного хука на глобальный уровень, благодаря этому:
- состояние модального окна полностью изолировано от основной страницы
- отсутсвуют ререндеры страниц или их элементов
- разделена бизнес логика, состояния и UI

## Почему именно zustand а не нативные способы?

- Современный стандарт
- При хранении состояния на странице была бы смешена логика состояний и логика UI
- При вынесе в кастомный хук вызывало ререндер страницы
- При создании компонента обёртки с кнопкой, количество модалок в DOM было бы пропорционально равно количеству элементов списка страницы
