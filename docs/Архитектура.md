## Стек
| Уровень           | Технология         | Почему                                  |
| ----------------- | ------------------ | --------------------------------------- |
| Фронтенд          | React + TypeScript | Стандарт индустрии                      |
| UI                | Mantine            | Готовые компоненты, тёмная тема         |
| API               | tRPC               | Типобезопасность, нет ручного fetch     |
| БД                | PostgreSQL         | Надёжность, транзакции                  |
| ORM               | Prisma             | Удобные миграции, типы                  |
| Состояние UI      | Zustand            | Лёгкий, без ререндеров                  |
| Состояние сервера | React Query        | Кеширование, инвалидация                |
| Типизация         | zod                | Общие типы пропсов на клиенте и сервере |

## Структура проекта
packages/  
├── server/ # tRPC + Prisma + Express  
├── client/ # React + Mantine + Zustand  
└── shared/ # Типы, Zod-схемы, утилиты

## Почему tRPC, а не REST
- **Типизация:** Изменил процедуру → фронт сразу видит ошибку
- **Меньше кода:** Нет `fetch`, ручных интерфейсов
- **Безопасность:** Zod валидация на сервере

## Почему Zustand, а не Redux
- **Легче:** 1KB против 10KB+
- **Проще:** Нет экшенов, редьюсеров
- **Точечная подписка:** Ререндерится только что нужно
