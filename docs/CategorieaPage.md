# CategorieaPage - Управление категориями

## Обзор

Компонент `CategorieaPage` является страницей управления категориями в приложении FinanceTracker. Он предоставляет интерфейс для создания, просмотра, редактирования и удаления категорий.

## Архитектура компонента

`CategorieaPage` состоит из трех основных компонентов:

1. **CategorieaPage.tsx** - основной компонент страницы
2. **CategoriesList.tsx** - компонент для отображения списка категорий
3. **CreateCategoriaModal.tsx** - модальное окно для создания/редактирования категорий

## CategorieaPage.tsx

### Импорты
- Использует компоненты из Mantine (`Button`, `useDisclosure`)
- Импортирует дочерние компоненты: `CreateCategoriaModal` и `CategoriesList`
- Использует React и типы из shared-пакета

### Функциональность
- Управляет состоянием модального окна с помощью `useDisclosure`
- Управляет состоянием редактируемой категории с помощью `useState`
- Предоставляет обработчики для открытия модального окна в режиме создания и редактирования

### Основные методы
- `onOpenCreate()` - открывает модальное окно для создания новой категории
- `onOpenUpdate(value)` - открывает модальное окно для редактирования существующей категории

## CategoriesList.tsx

### Импорты
- Использует компоненты из Mantine и Tabler Icons
- Использует tRPC для получения данных
- Использует кастомный хук `useDeleteCategory` для удаления категорий
- Импортирует типы из shared-пакета

### Функциональность
- Получает список категорий через tRPC запрос `trpc.categoties.getAllCategoryes.useQuery()`
- Отображает категории в таблице с возможностью редактирования и удаления
- Передает данные о выбранной категории в родительский компонент через callback `openUpdate`
- Обрабатывает удаление категории через `useDeleteCategory` мутацию

### Структура таблицы
- Колонка "Имя" - отображает название категории
- Колонка "Действия" - содержит кнопки редактирования и удаления

## CreateCategoriaModal.tsx

### Импорты
- Использует компоненты из Mantine (Modal, Form, TextInput)
- Использует кастомные хуки `useCreateCategory` и `useUpdateCategory` для создания/обновления категорий
- Импортирует схему валидации и типы из shared-пакета

### Функциональность
- Отображает модальное окно для создания или редактирования категории
- Использует Mantine Form для управления формой и валидации
- Использует схему валидации `createCatigories` из shared-пакета
- Автоматически определяет режим (создание или редактирование) на основе наличия пропса `category`
- Отправляет данные через соответствующую мутацию (создание или обновление)

### Пропсы
- `opened` - управляет видимостью модального окна
- `onClose` - callback для закрытия модального окна
- `category` - опциональный объект с данными редактируемой категории

## Поток данных

1. Пользователь заходит на страницу управления категориями
2. `CategorieaPage` отображает кнопку создания и список категорий
3. `CategoriesList` получает данные через tRPC и отображает таблицу
4. При нажатии на кнопку редактирования, `CategoriesList` вызывает `openUpdate` с данными категории
5. `CategorieaPage` передает эти данные в `CreateCategoriaModal` и открывает модальное окно
6. При отправке формы, `CreateCategoriaModal` определяет режим и вызывает соответствующую мутацию
7. После успешной операции данные автоматически обновляются благодаря tRPC

## Особенности

- Использует tRPC для типобезопасных запросов к серверу
- Использует Mantine для UI-компонентов
- Использует shared-пакет для общих типов и схем валидации
- Компонент `CategoriesList` обернут в `memo` для оптимизации ререндеров
- Названия переменных и функций содержат орфографические ошибки (например, `categoties`, `Catigories`), что является техническим долгом

## Зависимости

- `@mantine/core` и `@mantine/hooks` - для UI-компонентов и хуков
- `@tabler/icons-react` - для иконок
- `shared` - для общих типов и схем валидации
- `trpc` - для запросов к серверу
- `react` - для React-функциональности
